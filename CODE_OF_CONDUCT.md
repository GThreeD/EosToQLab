# Code of Conduct

## Our commitment

EosToQLab is committed to providing a welcoming, respectful, and harassment-free environment for everyone, regardless of experience level, identity, background, or role in the project.

Participation in this project means treating other contributors and users with patience, empathy, and professional respect.

## Expected behavior

Examples of positive behavior include:

- being considerate and constructive in technical discussions;
- assuming good intent while asking for clarification when needed;
- giving actionable feedback focused on the code, behavior, or documentation;
- respecting privacy, especially when logs or EOS show files may contain production information;
- acknowledging mistakes and helping repair their impact;
- accepting maintainer decisions gracefully, even when you disagree.

## Unacceptable behavior

The following behavior is not acceptable:

- harassment, discrimination, threats, or personal attacks;
- sexualized language or unwanted attention;
- deliberate intimidation, stalking, or sustained disruption;
- publishing another person's private information without permission;
- uploading confidential show data, credentials, passcodes, or other sensitive material;
- trolling, insulting comments, or repeated bad-faith arguments;
- any conduct that would be inappropriate in a professional working environment.

## Scope

This Code of Conduct applies in all project spaces, including issues, pull requests, code review, discussions, and other public or private communication related to EosToQLab.

It also applies when someone represents the project in public, for example by using an official project account or speaking on behalf of the project.

## Reporting

Report conduct concerns privately to the maintainer, **@GThreeD**, using a private contact method listed on the maintainer's GitHub profile.

Do not publish sensitive details in a public issue. If no private contact method is available and the behavior violates GitHub's policies, use GitHub's abuse-reporting tools.

Reports will be reviewed as promptly and impartially as possible. Information will be shared only with people who need it to evaluate and resolve the report.

## Enforcement

Project maintainers may remove, edit, or reject comments, commits, issues, pull requests, and other contributions that violate this Code of Conduct.

Depending on severity and history, responses may include:

1. a private clarification or correction;
2. a formal warning;
3. a temporary restriction from project interaction;
4. a permanent ban from project spaces;
5. escalation to GitHub or relevant authorities when necessary.

Retaliation against a person who reports a concern in good faith is itself a violation of this Code of Conduct.
